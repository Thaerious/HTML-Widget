NAME
    nidget - manage nidget based projects

SYNOPSIS
    nidget [OPTION]... [COMMANDS]...

OPTIONS
    -n, --name
        Set the name of the nidget when relevent

    -v, -vv
        Display verbose information, or very-verbose information

    --dist
        Set the 'distribution' directory for the dist command

    -o, --output
        Set location for locally deployed files (web server will look in this directory)

    -l, --link
        Create filesystem links instead of copying files

COMMANDS
    readme:  print the readme file which contains an example setup
    init:    create a new nidget project in the this directory
    create:  create a new nidget with specified name (-n)
    view:    create a new view with specified name (-n)
    records: show all records
    clean:   remove --dist and --output directories and remove records from nidget.json file

    babel:   translate .mjs files into .js placing them in the --dist directory, adds/updates records in nidget.json
    lib:     copy all 'dist' directories from modules to --output directory tree
    sass:    render .css files placing them in the --output directory
    pack:    browserify .js files for all the views, place in --output directory
    es6:     copy .mps files from source to --output [2]
    ejs:     render .ejs files to .html, place in --output directory

    dist  :  move .mjs and render .css files into the --dist directory for use as a module [3][4]
    deploy:  create deployable in --output directory for use by the web server [5]

NOTES
    [1] command alias' are 'scss' or 'css'
    [2] command alias 'mjs'
    [3] same as 'nidget sass ejs -o dist'
    [4] updates nidget.json file
    [5] same as 'nidget lib ejs sass es6'
